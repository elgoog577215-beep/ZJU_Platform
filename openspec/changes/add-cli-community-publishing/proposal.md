# 提案：为 AI 社区增加 CLI 投稿入口

## 背景

AI 社区已经收敛到统一内容池，技术分享和新闻热点都从 `/articles` 体系进入用户视野。部分同学习惯在本地用 Markdown、Word、PDF 或教程文件夹沉淀教程，如果只依赖网页编辑器，会产生复制格式、图片附件和重复登录等摩擦。

## 目标

- 提供一个账号密码登录的 CLI 投稿工具，让用户可以从本地上传技术分享和新闻热点。
- 服务端开放受认证保护的 CLI 投稿接口，复用现有账号体系、审核状态和内容表。
- 上传时必须选择栏目：`tech` 或 `news`。
- 技术分享落到文章内容池，新闻热点落到新闻内容池。
- 普通用户上传后进入待审核状态，管理员可以按现有规则直接发布或指定状态。

## 非目标

- 不开放匿名上传。
- 不绕过现有审核和权限模型。
- 不在本轮实现完整 Git 仓库同步、多人协作编辑或版本管理。
- 不把 AI 社区重新拆成新的独立教程系统。

## 影响范围

- 后端新增 CLI 聚合接口：登录仍复用 `/api/auth/login`，上传与发布走新的 CLI 受保护接口。
- CLI 新增 `login`、`whoami`、`publish`、`preview`、`status`、`logout` 命令。
- 文档导入能力扩展到 Markdown、Word、PDF、TXT、HTML 和教程文件夹的 `README.md`。
- 部署说明补充服务器反向代理、上传大小和认证访问要求。

## 风险

- 上传接口如果过宽，可能引入危险文件或垃圾内容，因此必须保留文件类型白名单、大小限制、登录校验和审核流。
- 新闻热点缺少来源会降低审核可信度，因此 CLI 发布 `news` 时必须提供 `source_url`。
- 文档解析无法保证所有格式 100% 保真，CLI 需要先给出预览能力，服务端返回解析元信息。
