## Why

The WeChat parsing assistant, event recommendation assistant, and event category filters currently maintain overlapping but separate category, audience, and keyword logic. This makes old WeChat parsing output drift from the newer event recommendation model, and forces the system to repair data after the model has already produced non-standard fields.

## What Changes

- Introduce a shared event intelligence catalog that defines the canonical event categories, aliases, audiences, campuses, and prompt context used by event-related AI workflows.
- Update WeChat event parsing so the model receives the canonical catalog up front and must directly return standard `category` and normalized audience values instead of legacy free-form labels.
- Add a shared event intelligence service for normalization, validation, confidence/reason metadata, and prompt-context generation.
- Update the event recommendation assistant to reuse the same catalog for intent detection, category labeling, and profile/category matching.
- Add a safe server-side classification script for existing `events` rows so legacy server data can be dry-run, backed up, and converted to canonical categories after deploy.
- Preserve backward compatibility for existing category filters by continuing to recognize legacy aliases during queries and classification.

## Capabilities

### New Capabilities
- `event-intelligence-catalog`: Shared canonical catalog and normalization contract for event-related AI workflows, including direct model extraction against standard options and safe legacy data classification.

### Modified Capabilities
- `event-ai-assistant`: Event recommendations must use the shared event intelligence catalog for category intent, labels, and scoring instead of private assistant-only taxonomy.
- `wechat-event-link-autofill`: WeChat event autofill must ask the model to return canonical event fields directly and validate them against the shared catalog before returning parsed data.

## Impact

- Backend utilities and services: new shared event catalog/intelligence modules under `server/src`.
- WeChat parsing: `server/src/utils/wechat.js` prompt and post-processing.
- Event assistant: `server/src/utils/eventAssistant.js` taxonomy and inference helpers.
- Data migration tooling: new `server/scripts/classify-event-categories.js` with dry-run/apply modes and database backup.
- API behavior stays compatible: event rows still use the existing `events.category`, `target_audience`, and `tags` fields; no destructive schema change is required.
- Rollback strategy: revert code changes and restore the backup generated by the classification script if an applied classification pass needs to be undone.
