## Context

站点当前使用 React、Vite 和 TailwindCSS 构建，公共视觉语言主要散落在 `src/index.css`、导航组件和各页面组件的 Tailwind 类名中。近期页面已经出现两种方向：一部分页面开始使用硬边框、强排版和块面结构；另一部分页面仍保留大圆角、玻璃拟态、强发光、渐变按钮和圆形操作按钮。

这次改造属于跨页面 UI 系统整理。它不改变接口、数据模型、路由和鉴权流程，只调整用户可见的视觉系统、布局节奏和高频交互控件形态。

## Goals / Non-Goals

**Goals:**

- 让站点形成统一的直角设计语言：硬边矩形、清晰边框、克制背景、稳定信息层级。
- 优先统一桌面导航、移动顶部栏、移动底栏，因为它们决定全站第一印象。
- 优先打磨活动页和视频页，因为它们保留了最多旧圆角卡片和圆形操作按钮。
- 让活动页在桌面和移动端更像信息发现工具，视频页更像内容资产展示页。
- 覆盖管理员登录、搜索、筛选等高频表单入口，让小控件不再暴露旧设计系统。
- 保留必要的语义圆形，例如头像、状态点、加载 spinner，不做机械清零。

**Non-Goals:**

- 不重构后端 API、数据库、鉴权、上传流程或内容数据。
- 不重写所有历史页面，不在本轮深度重构 `UploadModal.jsx` 的全部复杂编辑流程。
- 不引入新的 UI 库、图标库或外部设计依赖。
- 不把首页的视觉叙事推翻重做，只修正它与公共控件之间的风格冲突。

## Decisions

### Decision 1: 先建立 CSS 级直角设计令牌，再局部替换页面类名

全站会在 `src/index.css` 中补充直角系统工具类，例如直角表面、直角按钮、直角输入框、直角媒体卡、直角标签和移动导航块。页面组件优先复用这些工具类，再对特殊布局做少量 Tailwind 调整。

原因：当前组件中 Tailwind 类名数量多，如果只逐处替换 `rounded-*`，容易产生新的不一致。先定义少量稳定工具类，可以让导航、活动、视频、登录和筛选入口共享同一套边框、背景和交互状态。

替代方案是大规模抽新组件，但这会扩大风险，也容易碰到已有用户改动。本轮选择更轻的样式系统收束。

### Decision 2: 直角不是所有元素 0px，而是按语义分级

默认规则为：主面板、工具条、列表卡、弹层和按钮使用 `0-4px`；图片和媒体卡可保留 `4-6px` 的微圆角；头像、状态点、加载 spinner、真实圆形图标容器可保留圆形。

原因：完全 0px 在图片、移动端触控和中文大字场景下容易显得生硬。微圆角可以保留精致度，但不再形成旧版大圆角 App 气质。

### Decision 3: 活动页从卡片墙转成信息条目系统

活动页保留图片、标题、日期、地点、标签、评分、时长和操作入口，但统一为硬边列表卡。移动端优先阅读效率：图片固定尺寸，正文和元信息按稳定列排列，操作按钮弱化为边角入口。

原因：活动页是运营工具，用户要快速筛选和比较，不应被圆角大卡、发光标签和多个按钮抢注意力。

### Decision 4: 视频页从娱乐平台卡片转成硬边媒体资产展示

视频卡片改为硬边媒体块，减少遮罩厚度，把喜欢和打开操作改成硬边角落按钮。页面标题、排序和列表间距与活动页保持一致。

原因：视频页内容少时，大圆角媒体卡会显得空和旧。硬边媒体块可以与信息平台风格统一，同时更利于后续扩展为内容资产库。

### Decision 5: 导航先统一，弹窗入口轻量覆盖

桌面导航、移动顶部栏和底部导航会优先完成直角化。管理员登录、搜索和筛选入口使用同一套按钮、输入框和面板规则。上传弹窗只处理入口、底部按钮和明显破坏统一性的外层形态，不在本轮重写内部编辑器。

原因：导航和高频入口影响所有页面。上传弹窗文件体量大、状态多，贸然整体重构会扩大回归风险。

## Risks / Trade-offs

- [Risk] Tailwind 类名替换可能导致移动端按钮或文字溢出。
  Mitigation: 使用桌面与 390px 移动视口截图检查首页、活动页、AI 社区、视频页和管理员登录。

- [Risk] 清理发光和玻璃拟态后，页面可能短期显得更“冷”。
  Mitigation: 保留首页大图、AI 社区橙色主行动区和必要的品牌色块，用结构和色块提供记忆点。

- [Risk] mixed working tree 中已有改动可能与本轮样式改动混在同一文件。
  Mitigation: 只做与直角 UI 有关的局部修改，不重置、不删除、不提交，最终汇报明确涉及文件。

- [Risk] 上传弹窗只轻量覆盖，仍可能残留旧风格。
  Mitigation: 将上传弹窗深度统一列为后续专项，本轮只处理高频入口和明显外层冲突。

## Migration Plan

1. 新增直角系统工具类和主题变量补充。
2. 改造导航、移动顶部栏和底部导航。
3. 改造活动页桌面端和移动端列表、筛选和操作控件。
4. 改造视频页媒体卡、排序区和空白节奏。
5. 统一管理员登录、搜索、筛选等高频入口外观。
6. 运行 `npm run build`，再通过浏览器截图检查桌面和移动视口。

如出现回归，按组件文件回退对应视觉修改，不需要数据或接口迁移。
