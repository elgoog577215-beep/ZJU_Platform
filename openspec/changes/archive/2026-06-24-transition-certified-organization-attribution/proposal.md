# Why

平台当前由官方账号代录了大量学生社团、学生组织和学校官方部门活动。代录账号只说明“谁创建了记录”，不能说明活动真实属于哪个认证组织。随着组织后续入驻平台，需要把这些历史内容平滑过渡到对应认证组织名下，同时保留平台代录的审计事实。

# What Changes

- 新增认证组织历史归属迁移能力，管理员可以预览历史活动与认证组织的匹配候选。
- 匹配依据优先使用认证组织别名、主办方文本和来源信息；地点仅作为辅助证据，不单独决定归属。
- 管理员确认后才批量写入活动的公开发布主体和主办主体，不自动静默修改线上数据。
- 迁移保留原始上传账号、创建时间和历史审核事实，只新增或更新组织归属字段。
- 新增迁移日志，记录批次、匹配依据、置信度、确认人、变更前后主体和回滚状态。
- 后台增加一个轻量治理入口，用于查看候选、确认迁移和理解组织接管状态。

# Impact

- Frontend: 后台管理台新增认证组织归属迁移入口，展示候选预览、置信度和确认操作。
- Backend: 新增归属迁移服务和管理员 API，复用现有 `profiles`、`profile_aliases`、`ecosystem_partners`、`events`。
- Database: 新增迁移日志表；现有 `events.publisher_profile_id` 与 `events.organizer_profile_id` 继续作为第一阶段归属字段。
- Data Safety: 不修改 `events.uploader_id`、`created_at`、审核状态、文件路径；所有应用操作必须由管理员确认。
- Verification: 覆盖候选生成、强/中/弱匹配、批量应用、日志记录和不覆盖上传人的测试。
