# 提案：用户系统升级为账号与身份控制中心

## 背景

当前用户系统已经具备登录、个人主页、个人名片、活动画像、身份认证、组织 profile、投稿和成果认领等能力，但它们分散在不同标签和接口里。用户进入个人中心后，很难一眼判断自己的账号是否完整、拥有哪些身份、哪些组织主体可管理、哪些投稿或成果需要处理。

## 目标

- 把个人中心首屏升级为账号与身份控制中心，而不是只展示作品墙。
- 后端提供一个聚合接口，统一返回当前账号、资料完整度、身份认证、可管理 profile、内容状态和成果认领状态。
- 前端在个人中心展示资料完整度、待处理事项、可管理身份和快捷入口。
- 继续沿用现有 `users`、`profiles`、`profile_members`、`user_identity_claims`、投稿与成果表，不新增平行用户系统。
- 新增用户可见文案必须同步中英文。

## 非目标

- 不重写登录注册流程。
- 不新增组织入驻审批流。
- 不迁移历史内容归属。
- 不改变现有权限模型、审核模型或 token 存储策略。
- 不新增数据库表。

## 影响范围

- 后端新增 `/api/users/me/overview` 聚合接口。
- 前端 `PublicProfile` owner 视图新增账号系统总览。
- `src/services/api.js` 增加接口 helper。
- `public/locales/zh/translation.json` 与 `public/locales/en/translation.json` 增加总览文案。

## 风险

- 聚合接口读取多张表，必须保持只读、轻量，不在首屏引入重计算。
- 个人中心组件已经较大，本轮只做局部组件化，不进行大拆分。
- 资料完整度是产品提示，不应被当成权限门槛。
