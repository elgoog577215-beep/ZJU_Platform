# 影像库问题修复设计

## 目标

修复影像库当前可见的上传、分类、分页、后台展示与数据一致性问题，同时保持现有页面结构和视觉风格不做大重构。

## 范围

- 修复上传弹窗中误入的文章“代码”按钮。
- 防止上传弹窗在父组件重渲染时重置草稿。
- 为影像分类加载失败、加载中和空分类提供可见反馈。
- 为影像库照片和视频分别补充加载更多能力，避免旧内容被固定 limit 静默隐藏。
- 统一上传选择层的关闭、语义和状态边界。
- 修复视频 New 标识字段。
- 修正后台媒体分类列的“未分类”展示。
- 调整媒体分类计数，默认统计公开可见资源。
- 保留历史资源的分类名读取，避免软删除分类后历史资源变成“未分类”。

## 非目标

- 不重做影像库整体视觉风格。
- 不合并 photos 和 videos 数据表。
- 不引入新的上传存储方案。
- 不修改审核流和权限模型。

## 验收标准

- `npm run build` 通过。
- `/media` 页面可渲染。
- 上传图片或视频封面预览时不出现“代码”按钮。
- 分类接口失败时，影像库和上传弹窗都有明确提示。
- 照片和视频能分别继续加载后续页。
- 后台媒体分类列无分类时显示“未分类”。
