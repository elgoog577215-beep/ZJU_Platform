# Spec: ZJU AI 生态平台 UI 系统化升级

## Why

当前网站已有多个高质量视觉页面，但首页、关于页、活动页、AI 社区和后台的 UI 语言尚未完全统一。目标是让网站成为可信、现代、可运营的 AI 生态平台，而不是风格分散的页面集合。

## Visual Thesis

成熟高校 AI 生态操作台：沉浸页负责品牌势能，内容页负责高效发现与参与，社区页负责持续共建，后台负责稳定运营。视觉上使用克制的科技材质、清晰的信息密度和稳定的明暗主题。

## Goals

- 建立统一的产品级视觉目标：高校 AI 生态操作台。
- 统一首页、活动、社区、关于页的层级、材质、按钮、卡片和主题语言。
- 提升活动页和社区页的信息扫描效率。
- 保持首页/关于页的视觉冲击力，但避免文字拥挤和移动端压迫。
- 明暗主题都要像被专门设计过，而不是互相反色。

## Non-goals

- 不重构后端、数据库、AI 流程。
- 不一次性重做所有页面。
- 不改上传、登录、收藏、评论等业务行为。
- 不引入新 UI 框架或新依赖。

## Scenarios

### Scenario: 访客进入 AI 社区

GIVEN 用户从首页或导航进入 AI 社区
WHEN 页面加载完成
THEN 用户应能在第一屏理解这里包含技术分享、求助协作和社群入口，并能快速进入当前分区。

### Scenario: 用户浏览活动

GIVEN 用户进入活动聚合页
WHEN 浏览活动卡片和筛选区域
THEN 页面应更像高效内容产品，卡片信息层级清楚，控件语言和社区页保持一致。

### Scenario: 明暗主题切换

GIVEN 用户切换 day/dark mode
WHEN 查看首页、活动、社区、关于页
THEN 页面背景、文字、控件、卡片和状态提示都保持可读且有一致材质。

## Impact

- Frontend: `AICommunity.jsx`, `CommunityFeedPanel.jsx`, `CommunityTech.jsx`, `CommunityHelp.jsx`, `CommunityGroups.jsx`, `PostCard.jsx`, `Events.jsx`, `index.css`。
- Backend: 无。
- Database: 无。
- AI / standard data: 无。
- Deployment: 无。

## Risks

- 视觉改动可能影响移动端首屏高度；用 390x844 和桌面 1440x900 截图验证。
- 主题切换如果只改 DOM 标记会产生假截图；验证时必须通过 `ui_mode_v2` 或真实切换触发 React 状态。
- 社区共享面板改动会影响技术分享、求助、社群三个分区；需要逐页截图。

## Acceptance

- [ ] 首页、活动页、社区页、关于页在视觉上属于同一个产品。
- [ ] AI 社区第一屏明确呈现技术分享、求助、社群三条协作入口。
- [ ] 活动页卡片和筛选区更紧凑、可扫描，且保留现有筛选/AI 搜索行为。
- [ ] 桌面端无横向溢出、无明显重叠遮挡。
- [ ] 移动端首屏文字不压迫，关键操作可发现。
- [ ] day/night mode 都能读清楚，并有一致材质。
- [ ] `npm run build` 或相关 lint 通过，若失败需说明原因。
