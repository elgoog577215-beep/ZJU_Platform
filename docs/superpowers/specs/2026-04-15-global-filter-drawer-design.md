# Global Filter Right Rail Design

Date: 2026-04-15
Owner: Web Product/UI Team
Status: Draft for review

## 1. 背景与目标

当前站点大多数列表页均提供筛选能力，但筛选区常驻在主内容上方，常导致视觉切割明显。为提升内容阅读连续性，需要将筛选能力迁移到页面右侧空白区，在不牺牲筛选效率的前提下提升整体版面利用率。

目标：

- 全站列表页在桌面端统一采用“右侧常驻筛选栏（Right Rail）”模式。
- 主内容区优先，筛选从“上方大块区域”迁移至右侧空隙区域。
- 移动端采用右侧抽屉降级模式（不常驻）。
- 保持现有筛选字段与后端接口基本不变，优先 UI/交互重构。

## 2. 范围

### In Scope

- 所有用户侧列表页筛选交互改为“桌面右侧常驻栏 + 移动端右抽屉”
- 全站统一筛选分组、计数、应用/重置规则
- 全站统一“已选条件 chips 回显”
- 桌面/移动端统一交互语义与动效基线
- 可访问性与键盘交互规范

### Out of Scope

- 后端筛选语义重构
- 新增复杂查询 DSL
- 管理后台高阶筛选器深度改造（仅在第二阶段评估）

## 3. 信息架构与交互模型

### 3.1 页面结构

每个列表页统一由三部分组成：

1. 主工具条（主操作 + 排序）
2. 主体双栏区（左：列表内容，右：筛选栏）
3. 已选条件回显条（chips，可删除单项）

筛选明细不再占据主区上方，而是进入桌面右侧常驻栏。

### 3.2 桌面右侧常驻栏

- 右侧栏作为独立卡片容器常驻显示，不再依赖点击入口打开。
- 右侧栏宽度建议 `280-340px`，使用 `position: sticky`，`top: 24px`。
- 右侧栏包含筛选分组、已选数量、重置与应用操作区。

### 3.3 移动端降级行为（右抽屉）

- 打开方式：点击“筛选”按钮，从右侧滑入
- 关闭方式：右上角关闭按钮、点击遮罩、`Esc`
- 底部固定操作区：
  - 左：`重置`
  - 右：`应用`

### 3.4 分组布局

右侧筛选区（桌面常驻栏/移动抽屉）按分组组织：

- `排序`
- `分类`
- `标签`
- `时间`
- `状态`
- 业务扩展组（按页面需求插入）

规则：

- 高频组默认展开，低频组默认折叠
- 分组支持独立收起/展开
- 分组标题展示已选数量（可选）

## 4. 全站统一状态规则

### 4.1 状态定义

- `Pristine`：未修改
- `Dirty`：抽屉内已修改但未应用
- `Applied`：已应用到列表查询

### 4.2 按钮规则

- `应用` 按钮：仅在 `Dirty` 时高亮可点击
- `重置` 按钮：有任何已选条件时可点击
- 关闭抽屉时若 `Dirty`：
  - 默认丢弃未应用改动
  - 可选增加“是否放弃更改”确认（复杂页启用）

### 4.3 已选条件回显（Chips）

- 显示在主内容区顶部，单行优先，超出横向滚动
- 支持：
  - 删除单个条件
  - 一键清空
- 操作后立即触发查询（无需再打开抽屉）

### 4.4 空结果与错误处理

- 空结果：展示“当前筛选无结果”，附带“清空筛选”
- 请求失败：保留当前筛选状态，提示重试，不自动清空

## 5. 响应式与视觉规范

### 5.1 桌面端（右侧常驻栏）

- 布局采用 `minmax(0, 1fr) + 280~340px` 双栏结构
- 右栏 `sticky` 常驻，避免随滚动丢失筛选上下文
- 右栏卡片可叠加多个模块（标签、状态、时间、高级筛选）
- 主列表全宽用于内容阅读，不再被顶部筛选模块切断

### 5.2 移动端（右抽屉降级）

- 不保留常驻右栏，统一收敛为右侧抽屉
- 抽屉宽度：`88-92vw`
- 底部操作条固定，保证单手触达
- 重要操作区使用更大点击热区（>= 44px）

### 5.3 降噪原则

- 主区不放大块筛选面板
- 桌面利用右侧空白，不压缩主内容关键区
- 默认折叠低频分组
- chips 高密但不抢视觉主焦点

## 6. 可访问性（A11y）

- 桌面右栏为常规区域语义（非弹层），分组按钮可键盘切换
- 移动端抽屉使用 `role="dialog"` 与 `aria-modal="true"`
- 打开时焦点进入抽屉，关闭时焦点回到筛选入口
- 支持完整键盘路径：`Tab / Shift+Tab / Esc / Enter`
- 选中态与未选态满足对比度要求（WCAG AA）

## 7. 技术实现方案

## 7.1 新增通用组件

- `FilterRightRail`
  - 桌面常驻筛选容器，承载筛选分组与操作区
  - 提供 sticky 布局与分组折叠能力
- `FilterDrawer`
  - 仅移动端启用，负责遮罩、动效、焦点陷阱
  - 与右栏共享同一筛选内容组件
- `FilterSummaryBar`
  - 展示已选 chips
  - 支持单项删除与清空
- `FilterTriggerButton`
  - 移动端展示筛选入口与计数（桌面端可选仅作辅助入口）

### 7.2 状态管理建议

每页分两套状态：

- `draftFilters`（右栏/抽屉编辑态）
- `appliedFilters`（列表生效态）

流程：

1. 打开页面或打开移动抽屉时 `draft = applied`
2. 在右栏或抽屉内修改 `draft`
3. 点击应用后 `applied = draft` 并触发查询
4. 点击重置清空 `draft` 或 `applied`（按按钮语义）

### 7.3 与现有接口对接

- 保持现有 query 参数不变
- 前端统一映射层把 `appliedFilters` 序列化为接口参数
- 计数规则基于“非默认值字段数”

## 8. 迁移计划

### Phase 1（基建）

- 实现 `FilterRightRail` / `FilterDrawer` / `FilterSummaryBar`
- 在一个代表性页面试点（建议 AI 社区或文章列表）

### Phase 2（批量替换）

- 资源列表页（photos/music/videos/articles/events）统一迁移为右栏布局
- 社区列表（help/tech/groups/news rail）统一迁移为右栏布局

### Phase 3（回归优化）

- 统一动效、间距、排版
- 修复边界问题（空态、超长标签、多语言文案长度）

## 9. 验收标准

- 全站列表页桌面端默认展示右侧常驻筛选栏
- 移动端统一通过右侧抽屉打开筛选
- 筛选计数准确，chips 回显与实际查询一致
- 桌面右栏与移动抽屉均可稳定使用
- 键盘可完成筛选全流程
- 无构建错误与明显交互回归

## 10. 风险与应对

- 风险：各页面筛选字段差异导致统一组件失配
  - 应对：筛选内容区域 schema 配置化，容器层统一
- 风险：迁移期“新旧筛选并存”造成认知混乱
  - 应对：页面迁移时一次性移除旧筛选区
- 风险：计数规则不一致
  - 应对：沉淀统一 `countAppliedFilters()` 工具函数

## 11. 本次决策记录

- 方案选择：`A-RightRail - 桌面右侧常驻筛选栏 + 移动端右侧抽屉`
- 改造优先级：`All content lists`（全站列表优先）
- 移动端方式：`右侧侧滑面板（降级）`
