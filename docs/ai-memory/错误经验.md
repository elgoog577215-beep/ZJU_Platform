# 错误经验

## Events AI Search 跳转后页面空白

- 背景：修复英文模式下从 Events Hub 的 AI Search 切换到 Hackathon、Media Library 等页面后，URL 已变化但页面内容一片空白或加载不出来的问题。
- 触发：Playwright 复现发现点击导航后 `location.pathname` 已经变成目标路由，但 `#main-content` 里仍残留 Events 页面，且旧路由容器停在 `opacity: 0; transform: translateY(-8px) scale(0.996);`。
- 过程：一开始怀疑是黑客松页面懒加载、缓存、服务工作线程或返回关闭 hook。验证后发现目标页面可以直接打开，真正问题是全局路由层使用 `AnimatePresence mode="wait"`，Events AI Search 内部存在嵌套动画树时会拖住父级退出完成信号，导致新页面一直等待挂载。
- 结论：不要让全站路由切换依赖旧页面完整退出。当前项目已移除路由级 `AnimatePresence`，保留 `PageTransition` 的新页面入场动画，避免某个页面的子动画影响全站导航。
- 复用：如果以后出现“URL 变了但页面还是旧内容/空白”“页面容器 opacity 为 0 但未卸载”，优先检查路由级退出动画、嵌套 `AnimatePresence`、`mode="wait"` 和子组件退出动画是否卡住。
- 位置：`src/App.jsx`


## 2026-06-02 翻译资源被终端编码写坏

- 背景：跟进影像库、浙客松成果展示和报名页英文版时，需要批量补写 `public/locales/zh/translation.json` 与 `public/locales/en/translation.json`。
- 触发：修复后用户发现影像库和浙客松成果展示页被“修坏”。复现发现页面未崩溃，但中文模式大量文案显示为 `??`，英文成果页奖金符号 `¥` 显示为 `?`。
- 过程：一开始从组件运行时错误、路由和懒加载方向排查；Playwright 可见文本扫描显示错误集中在翻译资源值本身。继续检查 JSON 后确认，上一次通过 PowerShell here-string 内联 Node 脚本写入中文时，终端编码把中文和部分符号写成问号，导致中文资源被持久化破坏。
- 结论：在 Windows PowerShell 环境中，不能用包含中文正文的 here-string/内联脚本直接写翻译 JSON。修改中英文 i18n 资源时，应优先使用 `apply_patch`，或在脚本中用 Unicode escape 写中文值，并在写入后立即扫描 `??`、`????`、英文页 CJK 残留和关键符号如 `¥`。
- 复用：以后凡是修改 `public/locales/*.json`，必须做三项检查：`JSON.parse` 校验、目标页面中文资源问号残留扫描、Playwright 分别打开中英文关键页面做可见文本扫描。
- 位置：`public/locales/zh/translation.json`、`public/locales/en/translation.json`、`src/components/MediaLibrary.jsx`、`src/components/HackathonShowcase.jsx`。

## 2026-06-07 AI 社区内嵌播客深链参数冲突

- 背景：播客独立栏目被合入 AI 社区后，需要让旧 `/music?id=...` 链接跳到 AI 社区中的播客区域。
- 触发：全栈 Playwright 验证发现 `/music?id=1` 虽然进入了 `/articles`，但 AI 社区里的文章/帖子深链也使用 `id`，导致页面误开文章弹窗并把 hash 改成 `#modal-r11`。
- 过程：最初把旧音频链接映射为 `/articles?id=1#community-podcast`，这与 `CommunityTech` 等内容流的 `id` 语义冲突。修复时改为 `/articles?music=1#community-podcast`，同时保留 `/music?id=1` 到 `music=1` 的兼容转换。
- 结论：把独立资源栏目内嵌到复合页时，不能复用复合页已有的通用查询参数。音频深链使用 `music`，文章/技术分享继续使用 `id`，帖子使用 `post`，新闻使用 `news`。
- 复用：以后合并栏目或迁移深链时，先查目标页面所有 `useSearchParams` 消费的参数名，再设计专用参数，并用 Playwright 验证不会误开其他模块弹窗。
- 位置：`src/App.jsx`、`src/components/Music.jsx`、`src/components/AICommunity.jsx`、`src/components/SearchPalette.jsx`。

## 2026-06-29 历史回填：启动任务不能停在安装或命令执行

- 背景：早期启动本地网站时，依赖安装、前后端脚本和环境变量都可能部分成功，但用户真正要的是网站可访问。
- 触发：普通后台启动方式会在 shell 结束后被清理；缺少 `SECRET_KEY` 会让后端立刻退出；Codex App Node 会因 macOS 原生模块签名问题加载 `sqlite3` 失败。
- 过程：如果只看到命令执行过或依赖装完就汇报完成，会遗漏服务未持续运行、后端未健康、浏览器页面未打开这些真实失败。
- 结论：启动任务的错误模式是把内部步骤当最终状态。必须确认进程持续、HTTP 成功、健康接口正常、浏览器可见。
- 复用：后续启动/部署类任务不要用“命令已执行”替代“用户可访问”；遇到 macOS native module 报错，优先区分 Node runtime/签名问题和业务代码问题。
- 位置：`package.json`、`server/package.json`、`server/node_modules/sqlite3`

## 2026-06-29 历史回填：代码图谱不能替代源码和运行时验证

- 背景：codebase-memory 插件在前端清理中能快速定位组件，但用户关心它是不是真的可靠。
- 触发：图谱能给出架构视角和候选热点，却可能覆盖不到 JSX 细节、组件实际状态、浏览器运行效果或最新未索引改动。
- 过程：如果只依赖 graph 结论，就容易把“地图上的路径”误认为“真实代码行为”；历史中最终仍需要 `rg`、文件阅读、diff 和 DOM/浏览器验证。
- 结论：图谱过度信任是风险。它适合做第一层导航，不适合作为最终事实依据。
- 复用：后续使用图谱工具后，必须回到源码和运行时验证；组件搜不到或结果过粗时，快速切换 `rg`，不要反复搜索同一个图谱问题。
- 位置：`.codebase-memory/`、`docs/ai-memory/正向经验.md`

## 2026-06-29 历史回填：移动端 UI 清理不能只看单页截图

- 背景：移动端密度问题曾表现为多个页面各自“看起来有问题”，容易被当成一堆孤立视觉 bug。
- 触发：真实根因常常是桌面 spacing、骨架高度、移动工具条、卡片默认尺寸等共享模式遗留到手机端。
- 过程：如果只按截图逐页修，会产生大量小补丁，且共享组件继续把同类问题带到其他路由。
- 结论：移动端 UI 清理的错误模式是页面级追症状，不追共享根因。
- 复用：后续手机端审计先抽代表性路由，再找共享组件和重复密度模式；修后必须回到移动 viewport 复扫，不能只看 diff 或桌面。
- 位置：`src/components/MobileContentToolbar.jsx`、`src/components/SkeletonLoader.jsx`、`src/components/Events.jsx`

## 2026-06-29 历史回填：移动弹层问题不能只加 z-index

- 背景：ProjectPlaza 移动详情和海报弹层曾被底部导航或父容器影响。
- 触发：弹层位于带 transform 的路由容器内时，`position: fixed` 和高 z-index 仍可能受祖先 stacking/containment 限制。
- 过程：若只提高 z-index，会修不掉 transformed ancestor、scroll lock、底部导航遮挡、同功能兄弟弹层这些问题。
- 结论：移动弹层错误模式是把容器/portal 问题误判为单纯层级数字问题。
- 复用：后续弹层被裁剪、覆盖或无法全屏时，先查 portal 挂载点、祖先 transform、`MobileNavbar` 层级和 `100dvh`，再调 z-index。
- 位置：`src/components/ProjectPlaza.jsx`、`src/shared/ui/BodyPortal.jsx`

## 2026-06-29 历史回填：主题问题不能只调顶层遮罩

- 背景：白天模式黑块和动态壁纸不可见曾跨越多个层级，且用户明确希望用代码/DOM 对比而不是截图消耗。
- 触发：日间背景异常可能来自 active pill 深色类、fallback background、canvas clear color、overlay 白色遮罩或 modal/backdrop 多处叠加。
- 过程：只调一个 overlay 或截图猜测，会漏掉其他层导致“看起来修了但仍然黑/平/壁纸不可见”。
- 结论：主题/背景错误模式是只看最上层视觉表现，不查完整 background stack。
- 复用：后续 day/night 问题先对比分支和 DOM 样式，查 `BackgroundSystem`、theme token、fallback、overlay 全链路。
- 位置：`src/components/BackgroundSystem.jsx`、`src/components/Events.jsx`、`src/index.css`

## 2026-06-29 历史回填：规则评估不能用抽象考题代替真实场景

- 背景：`AGENTS.md` 对照实验最初用过偏抽象题，用户指出这类题不像真实协作问题。
- 触发：抽象题测出来的是格式控制和答题能力，不足以判断规则是否能改变真实项目行为。
- 过程：如果只用考试式问题评估，会误判规则有效性，也可能把“标题完整”当成“协作质量高”。
- 结论：规则/AI 行为评估的错误模式是脱离真实任务。
- 复用：后续评估规则、prompt 或 memory 体系，必须使用用户真实会问的项目场景，并保留原问题、分组条件和代表性回答。
- 位置：`docs/ai-memory/AGENTS对照实验报告-20260624.md`

## 2026-06-29 历史回填：AI 社区统一改造不能留下旧参数和死文案

- 背景：AI 社区合并四版面后，旧 `tab=groups`、解释性文案、右侧热榜语义等仍可能残留。
- 触发：社区信息架构从分散模块改成统一内容系统时，旧入口和翻译 key 如果不清理，会让用户再次看到旧心智。
- 过程：只改主组件会漏掉 `CommunityNewsRail`、用户中心 tab 路径、locale 死文案和旧路由重定向。
- 结论：信息架构合并的错误模式是只改主页面，不清理旧入口和旧语言。
- 复用：后续合并版面/栏目时，必须跨组件搜索旧参数、旧文案和旧入口，并验证用户中心/移动端路径不会擦掉 query。
- 位置：`src/components/AICommunity.jsx`、`src/components/CommunityNewsRail.jsx`、`public/locales/zh/translation.json`、`public/locales/en/translation.json`
