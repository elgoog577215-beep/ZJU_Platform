# 错误经验

## Events AI Search 跳转后页面空白

- 背景：修复英文模式下从 Events Hub 的 AI Search 切换到 Hackathon、Media Library 等页面后，URL 已变化但页面内容一片空白或加载不出来的问题。
- 触发：Playwright 复现发现点击导航后 `location.pathname` 已经变成目标路由，但 `#main-content` 里仍残留 Events 页面，且旧路由容器停在 `opacity: 0; transform: translateY(-8px) scale(0.996);`。
- 过程：一开始怀疑是黑客松页面懒加载、缓存、服务工作线程或返回关闭 hook。验证后发现目标页面可以直接打开，真正问题是全局路由层使用 `AnimatePresence mode="wait"`，Events AI Search 内部存在嵌套动画树时会拖住父级退出完成信号，导致新页面一直等待挂载。
- 结论：不要让全站路由切换依赖旧页面完整退出。当前项目已移除路由级 `AnimatePresence`，保留 `PageTransition` 的新页面入场动画，避免某个页面的子动画影响全站导航。
- 复用：如果以后出现“URL 变了但页面还是旧内容/空白”“页面容器 opacity 为 0 但未卸载”，优先检查路由级退出动画、嵌套 `AnimatePresence`、`mode="wait"` 和子组件退出动画是否卡住。
- 位置：`src/App.jsx`


## 2026-06-02 翻译资源被终端编码写坏

- 背景：跟进影像库、浙客松成果展示和报名页英文版时，需要批量补写 `public/locales/zh/translation.json` 与 `public/locales/en/translation.json`。
- 触发：修复后用户发现影像库和浙客松成果展示页被“修坏”。复现发现页面未崩溃，但中文模式大量文案显示为 `??`，英文成果页奖金符号 `¥` 显示为 `?`。
- 过程：一开始从组件运行时错误、路由和懒加载方向排查；Playwright 可见文本扫描显示错误集中在翻译资源值本身。继续检查 JSON 后确认，上一次通过 PowerShell here-string 内联 Node 脚本写入中文时，终端编码把中文和部分符号写成问号，导致中文资源被持久化破坏。
- 结论：在 Windows PowerShell 环境中，不能用包含中文正文的 here-string/内联脚本直接写翻译 JSON。修改中英文 i18n 资源时，应优先使用 `apply_patch`，或在脚本中用 Unicode escape 写中文值，并在写入后立即扫描 `??`、`????`、英文页 CJK 残留和关键符号如 `¥`。
- 复用：以后凡是修改 `public/locales/*.json`，必须做三项检查：`JSON.parse` 校验、目标页面中文资源问号残留扫描、Playwright 分别打开中英文关键页面做可见文本扫描。
- 位置：`public/locales/zh/translation.json`、`public/locales/en/translation.json`、`src/components/MediaLibrary.jsx`、`src/components/HackathonShowcase.jsx`。

## 2026-06-07 AI 社区内嵌播客深链参数冲突

- 背景：播客独立栏目被合入 AI 社区后，需要让旧 `/music?id=...` 链接跳到 AI 社区中的播客区域。
- 触发：全栈 Playwright 验证发现 `/music?id=1` 虽然进入了 `/articles`，但 AI 社区里的文章/帖子深链也使用 `id`，导致页面误开文章弹窗并把 hash 改成 `#modal-r11`。
- 过程：最初把旧音频链接映射为 `/articles?id=1#community-podcast`，这与 `CommunityTech` 等内容流的 `id` 语义冲突。修复时改为 `/articles?music=1#community-podcast`，同时保留 `/music?id=1` 到 `music=1` 的兼容转换。
- 结论：把独立资源栏目内嵌到复合页时，不能复用复合页已有的通用查询参数。音频深链使用 `music`，文章/技术分享继续使用 `id`，帖子使用 `post`，新闻使用 `news`。
- 复用：以后合并栏目或迁移深链时，先查目标页面所有 `useSearchParams` 消费的参数名，再设计专用参数，并用 Playwright 验证不会误开其他模块弹窗。
- 位置：`src/App.jsx`、`src/components/Music.jsx`、`src/components/AICommunity.jsx`、`src/components/SearchPalette.jsx`。
