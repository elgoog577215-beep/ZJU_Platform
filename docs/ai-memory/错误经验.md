# 错误经验

## Events AI Search 跳转后页面空白

- 背景：修复英文模式下从 Events Hub 的 AI Search 切换到 Hackathon、Media Library 等页面后，URL 已变化但页面内容一片空白或加载不出来的问题。
- 触发：Playwright 复现发现点击导航后 `location.pathname` 已经变成目标路由，但 `#main-content` 里仍残留 Events 页面，且旧路由容器停在 `opacity: 0; transform: translateY(-8px) scale(0.996);`。
- 过程：一开始怀疑是黑客松页面懒加载、缓存、服务工作线程或返回关闭 hook。验证后发现目标页面可以直接打开，真正问题是全局路由层使用 `AnimatePresence mode="wait"`，Events AI Search 内部存在嵌套动画树时会拖住父级退出完成信号，导致新页面一直等待挂载。
- 结论：不要让全站路由切换依赖旧页面完整退出。当前项目已移除路由级 `AnimatePresence`，保留 `PageTransition` 的新页面入场动画，避免某个页面的子动画影响全站导航。
- 复用：如果以后出现“URL 变了但页面还是旧内容/空白”“页面容器 opacity 为 0 但未卸载”，优先检查路由级退出动画、嵌套 `AnimatePresence`、`mode="wait"` 和子组件退出动画是否卡住。
- 位置：`src/App.jsx`
