# 工作流优化

## 2026-06-02 新增内容英文版同步

- 背景：跟进影像库、浙客松成果页英文翻译时，发现页面新增内容虽然主导航已英文，但部分静态数组、fallback 文案、空状态、SEO、ARIA 和按钮仍写死中文。
- 触发：用户明确要求以后凡是有新增内容都要自动跟进英文版。
- 过程：先用浏览器在英文模式下扫描 `/media`、`/hackathon`、`/hackathon/showcase` 的中文残留，再把页面文案接入 `public/locales/zh/translation.json` 与 `public/locales/en/translation.json`。
- 结论：任何新增用户可见内容都必须同步英文版，组件优先使用 `t(...)`，静态数组和 fallback 数据也要走翻译 key。
- 复用：后续修改页面、组件、空状态、Toast、SEO、ARIA、筛选字典或默认数据时，必须同时检查英文模式页面，避免只改中文。
- 位置：`AGENTS.md`、`public/locales/*/translation.json`

## 2026-06-02 macOS 本地前端验证运行时

- 背景：调整活动页右侧 AI 活动助手布局后，需要运行 lint、生产构建和活动助手 E2E。
- 触发：本机系统 PATH 没有 `npm`；使用 Codex App Node 或主运行时 Node 跑 `npm run build` 时，Rollup 原生模块 `rollup.darwin-arm64.node` 会因 macOS Team ID / library validation 不一致被拦截。
- 过程：先用独立 npm CLI 确认常规构建会失败，再复制 Raycast Node 到 `/tmp/zju-node-verify/node`，用 `codesign --force --sign - --entitlements` 加上 `com.apple.security.cs.disable-library-validation` 临时签名，然后直接调用 `node_modules/vite/bin/vite.js build`，绕过 npm script 重新切回 Codex App Node 的问题。Playwright 浏览器缺失时，用同一个临时 Node 执行 `node_modules/@playwright/test/cli.js install chromium`。
- 结论：这个仓库在当前 macOS 环境中，可靠验证命令是：
  - 单文件 lint：`./node_modules/.bin/eslint src/components/Events.jsx src/components/EventAssistantPanel.jsx e2e/event-assistant-flow.spec.js`
  - 构建：`/tmp/zju-node-verify/node node_modules/vite/bin/vite.js build`
  - E2E：`NO_PROXY=localhost,127.0.0.1,::1 /tmp/zju-node-verify/node node_modules/@playwright/test/cli.js test e2e/event-assistant-flow.spec.js --project=chromium`
- 复用：后续在本机遇到 `npm not found`、Rollup optional dependency “Cannot find module @rollup/rollup-darwin-arm64” 但 cause 是 `ERR_DLOPEN_FAILED` / Team ID mismatch 时，优先复用这套临时 Node 签名验证路径，不要误判为业务代码或依赖缺失。
- 位置：`docs/ai-memory/工作流优化.md`

## 2026-06-23 AGENTS 规则文档改动记录

- 结论：规则文档维护采用轻量方式：每次较大规则改动前先用时间戳备份 `AGENTS.md`；改动原因、范围和备份路径写入 `docs/ai-memory/工作流优化.md`。只有涉及新功能、多模块改造、数据库/架构/高风险变更时，才使用 OpenSpec。
- 复用：后续修改 `AGENTS.md` 时，先备份，再改动，再在本文件追加一条记录，说明改动背景、触发、过程、结论、复用和位置。
- 位置：`AGENTS.md`、`AGENTS.backup-20260623-005400.md`、`docs/ai-memory/工作流优化.md`
