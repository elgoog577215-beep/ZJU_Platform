# AGENTS.md

本文件约束在本仓库内工作的 AI / agent 的最小交付规范。默认目标不是“尽快改完”，而是“改完且可验证，不把回归留给下一个人”。

## 提交前必做

1. 先读实际代码和当前 diff，再下结论，禁止靠记忆描述现状。
2. 改动涉及 UI、交互、样式、弹层、响应式布局时，必须同时检查桌面端和移动端。
3. 改动涉及已有行为回归风险时，必须补测试或更新测试，不能只做手工验证。
4. 提交或推送前，必须运行与改动直接相关的测试/检查，并且结果通过。
5. 结果汇报中必须写明实际运行过的命令；不能说“应该没问题”。

## 最低验证要求

### 通用

- 对改过的前端文件，至少运行相关 `eslint`。
- 对新增或修改的测试，至少运行对应测试文件本身。

### UI / 布局改动

出现以下任一情况，必须补 Playwright 回归测试：

- modal / dialog / drawer / sheet / overlay
- 响应式断点差异
- 关闭按钮、返回按钮、层级遮挡
- 滚动容器、body scroll lock
- 移动端底部导航与浮层共存

这类测试至少覆盖：

- 移动端视口
- 桌面端视口
- 关键控件可见且未被遮挡
- 打开/关闭链路正常

## 不允许的提交

- 测试失败仍然提交/推送
- 明知没有验证移动端，却提交响应式 UI 改动
- 把无关生成物、无关 lockfile 噪音、临时调试文件一起提交

## 推荐命令

- `npx eslint <changed-files...>`
- `npx playwright test e2e/<relevant-spec>.js`

如果改动跨模块较大，再扩大测试范围，但至少先把最相关的测试跑通。
